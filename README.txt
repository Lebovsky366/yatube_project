Yatube — социальная сеть для публикации разной инфы

Yatube — это учебный проект социальной сети, где пользователи могут создавать учётные записи, публиковать записи, объединять их в тематические сообщества и управлять контентом через удобную админ-панель. Проект написан на Django с использованием Bootstrap для адаптивной вёрстки.

Используемые технологии: Python 3.9+, Django 2.2.19, Bootstrap 5 (локально или через CDN), SQLite (база данных), Git (система контроля версий).


Как развернуть проект на новом компьютере:

1. Клонирование репозитория:
   git clone https://github.com/ваш_username/yatube_project.git
   cd yatube_project

2. Создание и активация виртуального окружения:
   Для Windows (Git Bash):
     python -m venv venv
     source venv/Scripts/activate
   Для macOS / Linux:
     python3 -m venv venv
     source venv/bin/activate

3. Установка зависимостей:
   pip install -r requirements.txt

4. Применение миграций:
   cd yatube   # переходим в папку с manage.py
   python manage.py migrate

5. Создание суперпользователя (администратора):
   python manage.py createsuperuser
   Следуйте инструкциям: введите логин, email (можно пропустить), пароль.

6. Запуск сервера разработки:
   python manage.py runserver
   Проект станет доступен по адресу http://127.0.0.1:8000/

Основные страницы и URL:

- Главная страница: `/` (список последних 10 постов от новых к старым)
- Страница сообщества: `/group/<slug>/` (например, `/group/cats/`) — посты, принадлежащие определённой группе
- Админ-зона: `/admin/` (управление пользователями, постами и группами)

- Регистрация: `/auth/signup/`
- Вход: `/auth/login/`
- Выход: `/auth/logout/`
- Смена пароля: `/auth/password_change/`
- Восстановление пароля: `/auth/password_reset/`


Работа с админ-панелью:

После входа под суперпользователем по адресу `/admin/` доступны:
- Users – управление пользователями.
- Posts – список всех постов. Здесь можно добавлять, редактировать, удалять посты, а также назначать им группы.
- Groups – создание и редактирование тематических сообществ.
Для удобства в админке настроены: поиск по тексту постов, фильтрация по дате публикации, быстрое редактирование поля group прямо из списка постов.


Работа с моделями:

В проекте определены две основные модели.

Модель Post (пост):
- text – текст поста (TextField)
- pub_date – дата публикации (автоматически проставляется)
- author – ссылка на пользователя (ForeignKey)
- group – ссылка на сообщество, необязательное поле (ForeignKey, blank=True, null=True)

Модель Group (сообщество):
- title – название группы
- slug – уникальный адрес (часть URL)
- description – описание сообщества

После изменения моделей необходимо создать и применить миграции:
   python manage.py makemigrations
   python manage.py migrate


Регистрация и авторизация пользователей:

- Регистрация нового пользователя доступна по адресу `/auth/signup/`. Поля: имя, фамилия, username, email, пароль.
- После регистрации пользователь не входит автоматически – нужно зайти через `/auth/login/`.
- После входа в правом верхнем углу появляется имя пользователя и ссылка «Выйти».
- Выход осуществляется по адрему `/auth/logout/`. После выхода показывается страница с сообщением.
- Смена пароля доступна авторизованным пользователям по адресу `/auth/password_change/`.
- Восстановление пароля (если забыли) – по адресу `/auth/password_reset/`. Для тестирования настроена эмуляция почты: письма сохраняются в папку `sent_emails/` в корне проекта. Чтобы это работало, в `settings.py` указано:
   EMAIL_BACKEND = 'django.core.mail.backends.filebased.EmailBackend'
   EMAIL_FILE_PATH = os.path.join(BASE_DIR, 'sent_emails')


Поиск по постам:

На главной странице есть форма поиска (GET-параметр `q`). Поиск осуществляется по тексту постов без учёта регистра (`text__icontains`). Результаты отображаются в том же виде, что и обычные посты.


Страницы сообществ:

- При создании группы в админке указывается её slug (уникальный идентификатор, например `cats`).
- Посты могут быть привязаны к группе (необязательное поле `group`).
- Страница группы доступна по адресу `/group/<slug>/` и показывает последние 10 постов этого сообщества с подгрузкой авторов (`select_related`).


Настройка статики:

Статические файлы (CSS, изображения) хранятся в папке `static/` на уровне проекта. В `settings.py` прописано:
   STATICFILES_DIRS = [os.path.join(BASE_DIR, 'static')]

В шаблонах статика подключается через теги:
   {% load static %}
   <link rel="stylesheet" href="{% static 'css/bootstrap.min.css' %}">
   <img src="{% static 'img/logo.png' %}" alt="Логотип">

Часто возникающие проблемы и их решение:

1. Ошибка NoReverseMatch при ссылке на страницу группы:
   - Убедитесь, что в `posts/urls.py` есть `app_name = 'posts'` и `path('group/<slug:slug>/', ... name='group_list')`.
   - В шаблоне используйте `{% url 'posts:group_list' post.group.slug %}`.

2. На главной странице не отображаются посты:
   - Проверьте, что в `views.py` функция `index()` передаёт `posts = Post.objects.order_by('-pub_date')[:10]`.
   - Убедитесь, что в базе данных есть хотя бы один пост (создайте через админку).

3. Статика не загружается (нет стилей):
   - Проверьте, что папка `static/` находится в корне проекта (рядом с `manage.py`).
   - Добавьте `STATICFILES_DIRS` в `settings.py`.
   - Запустите сервер и проверьте доступность файла по адресу `http://127.0.0.1:8000/static/css/bootstrap.min.css`.

4. Ошибка 404 при переходе на страницу группы:
   - Убедитесь, что группа создана в админке и поле `slug` заполнено корректно (только латиница, цифры, дефисы).
   - Проверьте URL: `/group/правильный-slug/`.

5. Ошибка ConnectionRefusedError при восстановлении пароля:
   - Django пытается отправить письмо через реальный SMTP-сервер, но он не настроен.
   - Добавьте в `settings.py` строки:
        EMAIL_BACKEND = 'django.core.mail.backends.filebased.EmailBackend'
        EMAIL_FILE_PATH = os.path.join(BASE_DIR, 'sent_emails')
   - Перезапустите сервер. Письма будут сохраняться в папку `sent_emails/`.

6. После выхода не показывается страница с сообщением:
   - Убедитесь, что в `users/urls.py` для `logout/` используется собственная view-функция, которая вызывает `logout(request)` и рендерит шаблон `users/logged_out.html`, либо добавлен параметр `next_page=None` в `LogoutView.as_view()`.

7. Не отображается имя автора поста:
   - В шаблоне используется `{{ post.author.get_full_name }}`. Заполните у пользователя поля `first_name` и `last_name` через админку, либо замените в шаблоне на `{{ post.author.username }}`.


Примечания:

- Для оптимизации запросов к БД в `views.py` используется `select_related('author', 'group')`, чтобы избежать дополнительных запросов при обращении к связанным объектам.
- В проекте настроен русский язык интерфейса (`LANGUAGE_CODE = 'ru'`).
- Все страницы аутентификации (логин, логаут, смена пароля) используют кастомные шаблоны, оформленные с помощью Bootstrap.
